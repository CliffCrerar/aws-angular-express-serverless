About logins:

Conventionally it is good practice to use a JWT (Json Web Token) as the mechanism of client identification and authentication. This can be easily acheived by using Single Sign One. SSO is great because it allows a trusted third party to confirm authentication. The third parties can be google, facebook, microsoft. It is event possible to creat custom SSO credentials using cloud directory services like Azure Active Directory. All of the above mentioned providers use industry standard OAuth for authentication usually by the generation of JWT's. The final reason SSO is great is because it protects the proprioter of the application from any privacy issues that could arrise because no confidential info is shared by the third parties that provode the authentication means.
